# ACCESS SCORE ENGINE - DELTA UPDATE

## UPDATE 1: PRODUCT NOT FOUND OVERRIDE

### Rule

Before any FDA comparison or bucket calculation:

Compute:

```text
PolicyCoverageCount
```

using only FDA-comparison fields:

* Age
* BrandSteps
* GenericSteps
* Phototherapy
* TBTest
* SpecialistType

### Condition

If:

```text
PolicyCoverageCount = 0
```

Meaning all six policy fields are null.

### Action

Immediately return:

```json
{
  "bucket": 0,
  "reason": "PRODUCT_NOT_FOUND"
}
```

Stop further execution.

---

KnownFieldCount <= 1

# REAUTHORIZATION INFERENCE RULES

Apply before evaluation.

---

## Rule 1

If:

```text
ReauthorizationRequired IS NULL

AND

ReauthorizationDuration IS NOT NULL
```

Then:

```text
ReauthorizationRequired = Yes
```

Reason:

Presence of reauthorization duration implies reauthorization exists.

---

## Rule 2

If:

```text
ReauthorizationRequired = No

AND

ReauthorizationDuration IS NOT NULL
```

Then override:

```text
ReauthorizationRequired = Yes
```

Reason:

Duration information is stronger evidence than the No flag.

---

# CONSISTENCY CHECK

Apply only when:

```text
InitialAuthorizationDuration IS NOT NULL

AND

ReauthorizationDuration IS NOT NULL
```

### Rule

If:

```text
ReauthorizationDuration
<
InitialAuthorizationDuration / 2
```

Then:

```text
MinorRestriction += 1
```

Reason:

Access becomes significantly more restrictive after renewal.
